```markdown
# 과업지시서 (Statement of Work)

## 프로젝트 개요
- 프로젝트명: 도서 관리 API (book-mgmt-api)
- 발주처: ○○ 공공도서관
- 수행 기간: 2026-05-01 ~ 2026-07-31 (3개월 가상)
- 예상 규모: small

## 배경 및 목적
도서관의 장서 관리·회원 대출·반납 업무를 지원할 RESTful API 및 관리자 웹 화면을 구축한다. 기존 엑셀 기반 운영을 대체한다.

## 범위
### 포함 범위
- 도서 CRUD (등록/조회/수정/폐기) — API + 관리자 화면
- 회원 CRUD — API + 화면
- 대출·반납 처리 API
- 연체 알림 배치 (일 1회)
- 관리자 로그인 (ID/비밀번호)
- 도서 검색 (제목·저자·ISBN)

### 제외 범위
- 모바일 앱, 외부 결제, 타 시스템 연계

## 주요 요구사항 (개략)
### 기능
- 관리자 인증 및 권한(관리자/운영자 2단계)
- 도서 목록·상세·등록·수정·폐기
- 회원 등록·대출 이력 조회
- 대출(1인 5권 한도), 반납, 연체 알림
- 도서 검색 (키워드, ISBN)

### 비기능
- API 응답 시간 500ms 이내 (일반 조회)
- 동시 사용자 100명 수용
- 회원 개인정보(주민번호 마지막 4자리 제외한 모든 필드) 저장 시 암호화
- 로그인 실패 5회 초과 시 계정 잠금 (30분)

## 산출물 요구사항
스펙 §3-1 표준 산출물 전체 (분석·설계·시험·이행 + 설계감리 + 종료감리), 계층 구조(디렉토리+index.md+자식 파일) 준수.

## 제약사항·가정
- 단일 서버 배포 (scale=small)
- PostgreSQL 기반, FastAPI 예상
- Docker 단일 컨테이너
- 도서관 내부 네트워크에서만 접근

## 이해관계자
- 발주처: ○○ 공공도서관 정보화 담당자
- 수행사: 가상 SI 팀 (이 시스템)
- 감리: 외부 감리업체
```
